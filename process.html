From discovery to deployment

every part of the process I go through to create quality work is important.

Check out a little bit about how I get things done below.

Step One | Research, Discover, Plan
I start by making sure I know the scope and audience of the project inside out. Then I go all Nancy Drew on it and google it to death. This way I get to learn about the user I am designing for, the competition, and the market. This gives me a head start at understanding what works for the user and what doesn't.
Step Two | Brainstorm
I start brainstorming out all the obvious, and over done ideas to get them out of my head. This allows room for something amazing to start baking in my brain. I don’t force myself into any brainstorming structure because every project is so different. Sometimes I’m franticly forming words clouds; creating user personas, or sketching out thumbnail structures – whatever it takes to form concrete ideas.
Step Three | Blueprint, Information Architech
After I have some strong ideas and supporting information I begin to really blueprint out the ideas. I begin making user personas. Through making the personas a feature list almost makes itself. That feature list is the perfect place to start creating the Information Architecture. I make first a list stlye site map, and then a thumbnail site map. I sketch out thumbnails of the basic structure of each screen or page and then lay it out as a site map. This allows me to see the navigation structure as a whole and it will bring up any obvious navigation flaws right away.
Step Four | Wireframe and Mockup
From the visual site map I’ve made I begin focusing in on each thumbnail screen and creating low fidelity wireframes. Once I know the low fidelity wireframes work, and are approved I start of the high fidelity wireframes. Then I begin making the site or application mockups which is when everything starts coming to life.
Step Five | Prototyping
Once the mockups are done I begin making prototypes. Although prototyping can be introduced at almost point in the process (like paper prototyping) this is the most common place for me to begin interactive and create prototypes. The prototypes can be created in a number of different ways depending on the project type. Anywhere from an interactive PowerPoint, to a video animation or coded prototype - there are tons of different ways to bring the mockups to life.
Step Six | User Testing
Then user testing begins. This is one of the most crucial parts of the process and there is no lack of preparation. User testing allows for me to see any user experience flaws and fix them according to the learned behaviour of the user. The most common form of user testing I have executed is Co-discovery Learning. The method has proven very successful. User testing is the truest validation process there is. It tells us if we are ready to move onto development or if we need to go back to the drawing board.
Step Seven | Development Guidance
This is where my role shifts a bit. I step away from creating and instead guide the development team to ensure the features, functions and designs of the application or site are being executed exactly as they should be. There are always going to be small usability problems and tweaks that arise in the development stage that I want to make sure I answer. This way I get to see every project off from start to launch.
